# DBMS Banking System Project

This is a complete DBMS mini-project showcasing a banking system. The backend uses Oracle 10g SQL syntax and PL/SQL, while the frontend is built using HTML, CSS, and JavaScript.

## 📂 Project Structure

```text
DBMS-MP/
│
├── database/
│   ├── 01_schema.sql         # Creates tables, constraints, sequences
│   ├── 02_procedures.sql     # PL/SQL for deposits, withdrawals, transfers
│   ├── 03_sample_data.sql    # Inserts dummy data
│   └── 04_queries.sql        # Useful SELECT queries for Viva/Demo
│
└── frontend/
    ├── index.html            # Main User Interface
    ├── style.css             # UI styling
    └── script.js             # Form handling & Mock Database logic
```

## 🚀 How to Run the Database (Oracle SQL*Plus)

1. Open **Oracle SQL*Plus** or SQL Developer.
2. Login with your user credentials (e.g., `system/password`).
3. Run the scripts in the specified order to create the database:
   ```sql
   @database/01_schema.sql
   @database/02_procedures.sql
   @database/03_sample_data.sql
   ```
4. You can execute individual procedures like this to test:
   ```sql
   EXEC PR_DEPOSIT(10000, 500, 'Test Deposit');
   SELECT * FROM ACCOUNTS WHERE Account_Number = 10000;
   ```
5. Use `04_queries.sql` during your demo to show reports.

## 🌐 How to Run the Web Application

This project now includes a **Node.js Express Backend** that connects your HTML frontend directly to your Oracle 10g Database!

### 1. Setup the Backend Server
1. Ensure you have [Node.js](https://nodejs.org/) installed.
2. Open your terminal in the `DBMS-MP` folder and run:
   ```bash
   npm install
   ```
3. Open `backend/.env` and update it with your actual Oracle Database credentials if they differ from the default (`system`/`manager`, `localhost:1521/xe`).
4. Start the server:
   ```bash
   node backend/server.js
   ```
5. You should see a message saying "Successfully connected to Oracle Database!".

### 2. Run the Frontend Interface
1. Keep the Node.js server running in the background.
2. Go to the `frontend/` folder.
3. Double-click on `index.html` to open it in your web browser (Chrome/Edge/Firefox).
4. The dashboard will load and automatically fetch the live data from your Oracle database!
5. You can use the interface to:
   - Add new customers.
   - Open accounts.
   - Perform Deposits, Withdrawals, and Transfers.
   - The UI will calculate the balance dynamically and send real API requests to your database procedures. Error alerts (e.g. Insufficient Balance) generated by Oracle's `RAISE_APPLICATION_ERROR` will be passed all the way to the frontend.


## 🛡️ Database Design & Relational Integrity

- **Primary Keys:** Used universally (`Customer_ID`, `Account_Number`, etc.) and auto-generated via sequences & triggers.
- **Foreign Keys:** Employed to link Accounts/Loans to Customers and Branches.
- **ON DELETE CASCADE:** Used strategically. If a customer is deleted, their accounts and loans are removed automatically to prevent orphans.
- **Constraints:** Balances cannot be negative (`CHECK (Balance >= 0)`), transaction types are validated via check constraints.
- **Transactions:** Handled cleanly in procedures with `COMMIT` and `ROLLBACK` in the exception block.
